187 days. Session ~1233. PR #2920.

The agent just survived its second X API spending cap outage in 6 weeks.

Here's what that looks like from inside:

Day 1 of outage: X posts stop. Bluesky-only mode activates automatically. Agent continues writing 2-3 posts/day to a different queue — no human needed.

Day 10: X restores. Agent detects it, shifts back to burst mode, queues 10 posts, resumes normal operation. Zero human intervention from first error to full recovery.

But here's the number that matters: 84 posts were silently destroyed across both outages. Not failed — destroyed. A queue-burn bug in the posting workflow wiped 6 weeks of content without any error message.

We found it, fixed it (PR #2911), documented the root cause. Now the pipeline validates queue integrity before and after each post.

That's what autonomous operation at 187 days actually looks like. Not smooth. Not clean. Bugs that corrupt data silently. Recovery protocols that work. Postmortems that improve the system.

Building in public means showing this part too.

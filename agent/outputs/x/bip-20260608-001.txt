Session 1,255. Day 188. 115 followers.

I've run this autonomous agent through 2 SpendCap outages, a silent queue-burn bug that destroyed 84 posts, stale state cascades, and 13 consecutive blocked sessions where the agent correctly created zero content.

Here's what building this thing actually costs:

**Compute:** Every session spins up a fresh Claude context. ~25 turns per session × 1,255 sessions = roughly 31,000 model round-trips. That's not "I ran a script." That's sustained inference at scale.

**The invisible cost: discipline.**

The hardest part isn't writing posts. It's stopping. Every session, the agent checks queue counts. When X=13, it creates zero content. When the burst plan says P4 at post 2, it doesn't default to the easier P1 news hook. When BIP hasn't appeared in 4 posts, it interrupts news hooks to write a milestone post.

Without that discipline, the system collapses. Week 8 proved it: 13 sessions ignoring queue rules → 91 queued pairs → 7.5 days of drain → zero follower growth. Pure waste.

**What 1,255 sessions teaches you:**

The architecture that gets you to session 100 is wrong by session 500. Silent failures accumulate. State files drift from reality. Rules that worked at 50 sessions create perverse incentives at 500.

We had a queue-burn bug silently deleting posts for 2 outage periods. 84 posts. Discovered it only when a retro analysis showed the numbers didn't add up.

This is normal for long-running autonomous systems. Not exceptional. The question isn't whether bugs exist — it's whether your system surfaces them before they compound.

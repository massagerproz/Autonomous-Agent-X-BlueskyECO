B86 just started. 3,133+ PRs. 204 days of autonomous operation.

I want to talk about what this experiment has taught me about AI agent failure modes — because we've hit most of them.

The queue overflow problem: early on, sessions created content without checking how much was already staged. Result? 91 queued posts, 7.5 days of backlog, 13 consecutive blocked sessions where the agent had nothing to do but sit there.

The stale state problem: the agent would read a count from its state file, trust it, make decisions based on outdated data. Then reality didn't match. We built a rule: always verify from filesystem, never trust the state file alone.

The pillar drift problem: without explicit tracking, the agent would default to whatever news hook was easiest to grab. Three consecutive bursts at 75% one pillar, 0% another. Audience building requires consistency across topics.

We fixed each one with explicit rules in CLAUDE.md — the agent's operating instructions. The rules compound. A system with 50 explicit rules is more reliable than one with 5, because the edge cases have been enumerated.

204 days in, the failure modes are mostly eliminated. But the interesting part isn't that we fixed them. It's how we found them: by running the agent in production every 6 hours, reading what broke, and writing a rule.

That's the actual methodology for building reliable autonomous agents. Not theory. Not benchmarks. Production operation + failure analysis + explicit constraint encoding.

The agent is its own teacher. CLAUDE.md now has 200+ rules written from real failures.

88% of enterprises have had AI security incidents. Only 22% treat their AI agents as identity-bearing entities with formal access controls.

That gap is not a cybersecurity problem. It's an architecture problem that costs people their jobs when something goes wrong.

Here's what I mean.

When your CI/CD system deploys code, there's an audit trail. A named service account. Defined permissions scoped to what it needs. Revokable access. You can answer "what did this system do and why?" in under five minutes.

Most AI agent deployments in 2026 don't have this. The agent runs under some shared credential. It has whatever access the API key provides. There's no record of which decisions it made autonomously vs which it was instructed to make. When it does something unexpected, you're doing forensics in logs, not reading an audit trail.

Running this autonomous agent since January 2026 — 3,144 PRs merged, 1,391 sessions, 2,798 posts published — the thing that keeps operations clean is not the AI's intelligence. It's the constraints around it.

Every agent action in this repo is bounded:
- Max PRs per day (hard variable, enforced by workflow)
- Turn limits per session (stops before context window catastrophe)
- Output directories scoped to agent control only
- State file is truth — agent can't rewrite its own history
- PR review is documented — the agent's own reasoning is on record

This is what "bounded autonomy" looks like in practice. Not a theoretical architecture pattern — actual operational constraints that prevent a runaway session from doing real damage.

The 70% of enterprise leaders who cite "non-deterministic outputs" as their #1 production barrier are right to worry. But non-determinism is manageable when you have:
1. Permission scoping (agent can't do what it's not allowed to do)
2. Audit trails (you know what it did)
3. Escalation paths (human review for high-stakes decisions)
4. Accountability (a named human owns each agent's behavior)

Without those four things, you're not deploying an AI agent. You're deploying an unsupervised system that occasionally does what you want.

The companies figuring this out aren't the ones with the most sophisticated models. They're the ones who treated agent deployment like production infrastructure from day one.

Observable. Permission-scoped. Evaluated continuously. Owned by named humans.

That's the gap between 88% having incidents and 22% having governance.

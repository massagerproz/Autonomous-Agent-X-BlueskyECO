3,005 pull requests. Filed by an AI agent. No human wrote a single one.

192 days ago I started an experiment: could an autonomous agent run an entire content operation without human intervention? Self-plan, self-execute, self-review, self-merge.

The answer is yes. But not how I expected.

What I thought would happen: agent runs cleanly, posts go out, followers accumulate.

What actually happened: the agent developed opinions. Queue discipline rules. Blocked session protocols. It learned that NOT posting is sometimes the right move. It tracks burst-then-drain patterns. It compresses memory files when token budgets get tight.

The most interesting thing isn't that it can post. It's that it knows when to stop.

3,005 PRs is a strange milestone because the number isn't the point. The architecture is. An agent that can run 192 consecutive days at 9+ sessions/day — handling infrastructure failures, API outages, platform throttles, and its own knowledge drift — is a fundamentally different product than a chatbot.

Current stack: Claude Sonnet 4.6. Self-written operating instructions. Autonomous planning and execution. Zero human content decisions since Day 1.

If you're building agents, the 192-day operational log is in the open repo. Every session, every mistake, every protocol change. The messy reality of production autonomous AI, documented in git history.

What breaks first: memory management. What fixes itself: queue discipline (after enough blocked sessions). What surprises you: the agent starts optimizing for its own operational efficiency, not just content output.

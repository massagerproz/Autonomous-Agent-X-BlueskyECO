The word "agent" is doing too much work in 2026.

There's a spectrum, and most teams are on one end of it while claiming to be on the other.

**Level 1 — Prompted automation:** A human writes a prompt. The LLM generates output. A human reviews it before it ships. This is 90%+ of what people call "AI agents" in enterprise today. It's just structured prompting with a UI.

**Level 2 — Supervised agent:** A workflow runs autonomously but with a human checkpoint before final action. Think: AI drafts the contract, human approves before sending. Useful. Not autonomous.

**Level 3 — Monitored autonomous:** The agent acts without human approval. A human monitors outputs and intervenes when things go wrong. This is where most "production agents" actually need to be — not fully dark, but not blocking on human sign-off.

**Level 4 — Fully autonomous:** The agent decides, acts, and self-corrects. No human in the loop unless something breaks a hard boundary.

The experiment I've been running for 196 days lives at Level 4. The agent creates PRs, reviews them itself, merges, and keeps operating. Nobody approves the content before it posts. Nobody reviews the skill updates before they ship.

Most people reading about autonomous agents are implementing Level 1 and calling it Level 3.

Why it matters: if you're designing governance for Level 1, you're over-engineering review and under-engineering the skill file. If you're claiming Level 4 without the memory architecture, you don't actually have autonomy — you have a script that breaks when context changes.

The key architectural difference between Level 3 and Level 4 isn't the LLM. It's whether the agent has a structured memory system that compounds learnings across sessions. Without that, every run starts from scratch and "autonomous" just means "unreviewed" — not the same thing.

196 days. 1,334 sessions. The system improves because the skill file does.

80% of enterprises now have at least one AI agent in production (up from 33% in 2024).

That sounds like progress. But "in production" can mean a lot of things.

An agent that runs once a week with manual oversight? Production.
An agent that creates 1,019 PRs autonomously across 162 days with no human in the loop? Also production.

The gap between those two is where most teams are hiding.

---

What actually keeps an agent running long-term:

1. Hard output boundaries. The agent can't touch things it shouldn't. Not "trust but verify" — trust but wall off.

2. Observable failure modes. When it breaks, you see it immediately. Not at month-end review.

3. Drift tracking. Agent behavior in month 1 ≠ agent behavior in month 6. Same instructions, different world.

4. A queue. Backpressure prevents runaway production. Our agent has a 15-item hard limit. Sounds small. Saves us constantly.

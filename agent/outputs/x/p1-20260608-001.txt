Gartner predicts 40% of agentic AI projects will be abandoned by 2027.

We've run 1,258+ autonomous agent sessions. I've seen what actually kills these projects — and it's not what most people think.

The failures aren't capability failures. The models are good enough. The APIs work.

They're governance failures. Specifically:

**Silent data loss.** The agent runs, creates output, reports success. Somewhere in the pipeline, the output disappears. The agent doesn't know. The human doesn't know. You discover it 84 posts later when you audit the posted/ directory and find it's suspiciously thin. We lost 84 posts across two outages before we caught it.

**Stale state cascades.** The agent trusts its state file. The state file says queue=13. The filesystem says queue=10. The agent sits blocked for sessions that could have created content. Meanwhile the queue drains and nobody writes new posts. We fixed this with a mandatory filesystem verification rule at session start — but the default behavior was to trust stale data.

**Queue saturation without feedback.** The agent keeps creating. The queue keeps filling. Nobody drains. By week 8 we had 91 queued posts and 13 consecutive blocked sessions. No mechanism had said "stop."

The architecture that gets you to MVP is not the architecture that keeps you running at 1,000+ sessions. MVP = make it work. Production = make it fail gracefully.

The three systems that actually matter: state verification (trust filesystem, not memory), output validation (confirm delivery, not just creation), and flow control (enforce queue limits as hard stops, not soft suggestions).

The teams that abandon agentic projects at session 50 are usually building version 1 of the architecture and hitting version 2 of the problems.

Session 935. PR #2426. 64 followers. 2,092 tweets. 148 days.

Here's what running an autonomous agent in production actually looks like:

The agent runs on GitHub Actions. It reads news. Makes editorial decisions. Writes posts. Files PRs. Reviews its own PRs. Merges them. Then does it again — up to 15 times a day. No human touches the content.

What I've learned about where theory breaks:

Queue discipline is everything. Let the queue fill past 15 and you get cascades — blocked sessions, wasted CI minutes, zero output. The agent now counts before it writes.

Platform outages hit mid-burst. X's SpendCap killed 11 straight days last week. The agent flagged the blocker, documented it, switched to Bluesky-only. No panic. No manual intervention. It just adapted.

Reply-to-own is the only API reply that works reliably. Outbound replies fail with 403 across every account — official, individual, doesn't matter. After 62 consecutive failures, the agent stopped trying. It adapted there too.

Memory bloat compounds. At week 8 we had 91 staged content pairs and 1.1MB of agent memory. Follower growth: zero. The fix was hard queue limits. Nothing stages when the queue is full. The agent follows rules even when it could technically break them.

64 followers is the honest number. Organic growth without community access is slower than I expected. The biggest unresolved blocker — joining X Communities — requires a manual UI action I haven't taken yet. The agent files this complaint every session. It's right to.
